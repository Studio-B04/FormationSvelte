Les adapters officiels

adapter-static

Cet adapter permet de convertir votre application en fichier statiques (HTML, CSS, JS) pour une utilisation sur un serveur Web statique comme OVH ou Firebase par exemple.

```js import adapter from '@sveltejs/adapter-static'; export default { kit: { adapter: adapter({ // adapter options go here }) }, }; ```

adapter-node

Cet adapter permet de builder votre application en une application Node. Le point d'entrée ne sera donc plus un fichier HTML mais un fichier JavaScript.

Cet adapter permet de créer une applcation qui peut être deployé sur un serveur Node comme AWS, Google Cloud, Github Pages, etc.

```js import adapter from '@sveltejs/adapter-node'; export default { kit: { adapter: adapter({ // adapter options go here }) }, }; ```

adapter-netlify

Netlify est une plateforme d’hébergement et d’automatisation de projets gratuite (certaines options sont payantes)

L'adapter adapter-netlify va convertir l'application en fichiers pour Netlify.

```js import adapter from '@sveltejs/adapter-static'; export default { kit: { adapter: adapter({ // adapter options go here }) }, }; ```

Une configuration différente en fonction de l'environnement

Il peut arriver que pendant la phase de développement de l'application, vous ayez besoin d'un serveur spécifique (Un serveur dédié à la recette, un serveur pour les test d'intégrations, etc.)

On peut alors tout à fait configurer un adapter différent pour le dev et la prod :

```js import adapterNode from '@sveltejs/adapter-node'; import adapterStatic from '@sveltejs/adapter-static'; const isProd = process.env.NODE_ENV === 'production'; const adapter = isProd ? adapterNode : adapterStatic; export default { kit: { adapter: adapter() } }; ```

L'adapter-node sera utilisé en prod et l'adapter-static sera utilisé pour la phase de développment.
