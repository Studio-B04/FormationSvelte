Organisation des fichiers

Si l'installation s'est déroulée comme prévu, vous devriez avoir des dossiers et des fichiers dans votre repertoire /formationSvelte. Voici les plus important :

``` html {*}{lines:false} 📁 formationSvelte └─── 📁 .svelte-kit │ └─── ... └─── 📁 src │ └─── 📁 lib │ └─── 📁 routes │ └─── 📃 +page.svelte │ └─── 📃 app.html └─── 📁 static │ └─── ... ... └─── 📃 svelte.config.js └─── 📃 vite.config.js ```
--- layout: default transition: slide-up ---

svelte.config.js

Ce fichier permet de configurer votre projet SvelteKit, et notamment les adapters.

Les adapteurs sont des utilitaires qui permettent de builder votre projet en fonction de la plateforme sur laquelle il sera deployé (Cloudflare, Netlify, Node, Vercel, static, etc.)

```js import adapter from '@sveltejs/adapter-auto'; /** @type {import('@sveltejs/kit').Config} */ const config = { kit: { adapter: adapter() } }; export default config; ```
--- layout: default transition: slide-up ---

vite.config.js

Ce fichier permet de gérér la configuration de Vite.

Vous pouvez y créer des alias de dossiers, changer le réprtoire de dev ou de build, ajouter des plugins, etc.

```js import { sveltekit } from '@sveltejs/kit/vite'; import { defineConfig } from 'vite'; export default defineConfig({ plugins: [sveltekit()] }); ```
--- layout: default transition: slide-up ---

app.html

C'est le point d'entrée de votre application. Toutes les pages seront basées sur ce fichier de base.

```html %sveltekit.head%
%sveltekit.body%
```

Le dossier /static

C'est le dossier qui contient tous les assets de votre projet, accessibles depuis le front.

Vous pouvez y stocker des images, des videos, des fichers CSS, etc. Ils seront accessibles depuis vos composant à la racine de votre projet.

Exemple :

```html └─── 📁 static └─── 📁 images └─── 📃 photo.jpg ```


Le dossier /src

C'est le dossier qui contient toutes les sources de votre projet et notamment les routes. Vous etes libres d'y créer autant de sous dossiers que vous souhaitez, et créer des alias dans le fichier svelte.config.js si besoin. Exemple :

```html {*}{lines:false} └─── 📁 src └─── 📁 utils └─── 📃 date.js ```
```js /** @type {import('@sveltejs/kit').Config} */ const config = { ... kit: { alias: { '@utils': './src/utils/' } } }; ```
```svelte ```

Créons un dossier /components

Dans le dossier /src, nous allons créer un sous-dossier components, qui contiendra tous les composants de notre application, et créer l'alias correspondant :

```js /** @type {import('@sveltejs/kit').Config} */ const config = { ... kit: { alias: { /* vous pouvez nommer l'alias comme vous le souhaitez ! */ '@components': './src/components/' } } }; ```
