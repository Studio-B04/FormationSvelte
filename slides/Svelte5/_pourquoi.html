À quoi ça sert ?

On pourrais penser que tous les changements de syntaxe que l'on a vu n'apportent pas grand chose puisqu'il s'agit simplment d'une façon différente de faire la même chose.

En réalité ils apportent une plus grande clarté au code, et une simplification des applications. En effet il est moins souvent nécéssaire d'avoir recours à des stores ou des événements custom pour gérer des états complexes.

Mais la grande force des Runes, c'est qu'elle permettent de dissocier les composants de leur logique.

--- layout: default transition: slide-up ---

Voici un exemple

Prenons en exemple ce composant Counter.svelte :

```svelte

{title}

```
--- layout: default transition: slide-up ---

Voici un exemple

On pourra alors créer un fichier, que l'on appellera counter.svelte.js et qui contiendra la logique de notre composant :

```js export function createCounter() { let count = $state(0); let double = $derived(count * 2); const increment = () => { count ++ }; $effect(() => console.log('counter was updated to : ', count)); return { increment, get count () { return count; }, get double () { return double; } } } ```
--- layout: default transition: slide-up ---

Voici un exemple

Le composant Counter.svelte pourra ainsi être récris comme ceci :

```svelte

{counter.title}

```

Notez que pour conserver la réactivité, counter ne doit pas être déstrucuré !

```js {*}{lines:false} const { title, count } = createCounter(); ```
```js {*}{lines:false} const counter = createCounter(); ```
